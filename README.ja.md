# IMSLP Accompanist

IMSLP などのパブリックドメイン楽譜 PDF から**伴奏パート（主にピアノ）**を抽出し、ブラウザで再生できる Web アプリのプロトタイプです。

> English version: [README.md](./README.md)

## このアプリでできること

### 1) 楽譜解析
- PDF をアップロードし、**Audiveris** で OMR を実行して MusicXML と小節バウンディングボックスを抽出します。
- 伴奏パート（主に 2 段譜のピアノ）を自動判定します。
- 非伴奏パートから音符密度ヒューリスティックでソロパートを自動選定します。
- テンポは多段フォールバックで推定します（`<sound tempo>` -> `<metronome>` -> イタリア語テンポ語 -> PDF 上部 OCR -> 120 BPM）。
- タイトルは MusicXML メタデータ優先で、未取得時に OCR へフォールバックします。
- PDF と一緒に手元の MusicXML をアップロードすると、そのデータを優先し、OMR をスキップできる場合があります。
- MusicXML 単体アップロードにも対応しています（PDF 連動ハイライトなし）。

### 2) 再生機能
- 伴奏を Salamander Grand Piano（Tone.js サンプラー）で再生します。
- ソロパート検出時はバイオリン風シンセを同時再生します。
- ソロ音量モード：通常 / カラオケ（小さめ） / ミュート。
- 伴奏音量（0〜120%）・テンポ（30〜240 BPM）をリアルタイム調整できます。
- 開始小節・終了小節、ループ、カウントイン（0〜4 小節）、メトロノームに対応しています。
- フェルマータを考慮し、音価延長中のメトロノームを抑制します。

### 3) 表示機能
- **PDF ビュー**：元 PDF 上で再生中の小節をハイライト表示します。
- **譜面ビュー**：OSMD で MusicXML を描画し、現在小節へカーソル追従します。
- OMR 出力を安定表示するための MusicXML サニタイズ（`time`、`divisions`、不正ノート、欠落小節、調号整合など）を実施します。
- 解析済み MusicXML をダウンロードし、次回以降の高速再利用が可能です。

## 起動方法

### 方法 A（推奨）：Docker Compose

```bash
docker compose up --build
```

- フロントエンド: <http://localhost:5173>
- バックエンド: <http://localhost:8000>（`GET /health`）

> 初回ビルドは Audiveris 関連イメージが大きいため時間がかかることがあります。

### 方法 B：フロント/バックを個別起動

#### バックエンド
```bash
cd backend
pip install -e .
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

#### フロントエンド
```bash
cd frontend
npm install
npm run dev -- --host 0.0.0.0
```

## 使い方（最短手順）
1. <http://localhost:5173> を開きます。
2. 楽譜 PDF をドラッグ＆ドロップします。
3. 解析完了まで待ちます。
4. 再生ボタンで伴奏再生と小節ハイライトを確認します。
5. テンポ、ループ、カウントイン、メトロノーム、音量設定を調整します。
6. 譜面ビューでカーソル追従を確認します。
7. 必要なら MusicXML をダウンロードし、次回の高速解析に使います。

## 認識精度に関する重要事項
- 楽譜読み込み精度は **Audiveris（外部ソフトウェア）に強く依存**します。
- OMR の性質上、**常に正確な読み込みを保証することは難しい**です。
- 精度を上げるため、可能であればスキャン楽譜よりも**打ち込みの綺麗な楽譜（デジタル浄書譜）**の利用を推奨します。
